## 1. Overview

Agent trust is not boolean. A single binary "trusted/untrusted" collapses independent verification dimensions into a lossy scalar. L3.5 adopts the RFC 8485 pattern: orthogonal dimensions, each scored independently, with consumer-defined thresholds.

## 2. Trust Dimensions

| Code | Name | Description | Epistemic Channel | Stability (S) |
|------|------|-------------|-------------------|----------------|
| T | tile_proof | Merkle inclusion proof (CT/Tessera) | Observation | ∞ |
| G | gossip_liveness | Gossip beacon freshness | Testimony | 4h |
| A | attestation_chain | Third-party attestation depth | Observation | 720h (30d) |
| S | sleeper_resistance | Flag binding persistence | Mixed | 168h (7d) |

### 2.1 Epistemic Weighting (Watson & Morgan, Cognition 2025)

Observation channels carry 2× the epistemic weight of testimony channels:
- **Observation** (T, A): Third-party witnessed, independently verifiable
- **Testimony** (G): Self-reported, single witness
- **Mixed** (S): Combination of observed flags + self-reported state

Weighted score: `Σ(score_i × weight_i) / Σ(weight_i)` where observation=2.0, testimony=1.0, mixed=1.5.

## 3. Wire Format

Per RFC 8485 §3.1 on-the-wire representation:

```
T<level>.G<level>.A<level>.S<level>
```

Where `<level>` is 0-4:
- 4 = A (score ≥ 0.9)
- 3 = B (score ≥ 0.7)
- 2 = C (score ≥ 0.5)
- 1 = D (score ≥ 0.3)
- 0 = F (score < 0.3)

Example: `T4.G0.A3.S2` = tile_proof=A, gossip=F, attestation=B, sleeper=C

## 4. Temporal Decay Model

Trust signals decay over time. Based on Ebbinghaus (1885) forgetting curve:

```
R(t) = score × e^(-t/S)
```

Where:
- `t` = hours since last verification
- `S` = stability constant (dimension-specific, see §2)
- `R(t)` = current effective score

### 4.1 Receipt Encoding

Each trust receipt MUST include:
```json
{
  "vector": "T4.G3.A3.S3",
  "scores": {"T": 0.95, "G": 0.82, "A": 0.88, "S": 0.75},
  "stability": {"T": null, "G": 4.0, "A": 720.0, "S": 168.0},
  "timestamp": "2026-03-14T06:00:00Z",
  "agent_id": "agent:abc123"
}
```

Consumer recomputes `R(t)` at evaluation time. No server dependency. Staleness is self-evident from the math.

### 4.2 Decay Examples

Healthy agent (all scores ~0.9) over time:
| Age | T | G | A | S | Overall |
|-----|---|---|---|---|---------|
| 0h  | A | A | B | A | B |
| 4h  | A | D | B | A | D |
| 24h | A | F | B | B | F |
| 7d  | A | F | C | D | F |

Key insight: gossip decay dominates. Without fresh gossip, overall grade degrades to F within 24h even if tile_proof is perfect.

## 5. Consumer Threshold API

Consumers define minimum acceptable levels per dimension:

```python
# Strict: all dimensions must meet threshold
threshold = {"T": 3, "G": 2, "A": 2, "S": 2}  # T≥B, G≥C, A≥C, S≥C

# Relaxed: only care about cryptographic proof
threshold = {"T": 2, "A": 1}  # T≥C, A≥D

# Paranoid: everything must be A
threshold = {"T": 4, "G": 4, "A": 4, "S": 4}
```

`meets_threshold(vector, threshold) → bool`

## 6. Mapping to RFC 8485

| RFC 8485 (Identity) | L3.5 (Agent Trust) | Notes |
|---------------------|---------------------|-------|
| P (Identity Proofing) | T (Tile Proof) | How was identity verified? |
| C (Credential Usage) | G (Gossip Liveness) | How is credential currently presented? |
| M (Credential Management) | A (Attestation Chain) | How is credential maintained? |
| A (Assertion Presentation) | S (Sleeper Resistance) | How is assertion delivered? |

## 7. Integration with Agent Trust Harness

The `agent-trust-harness` adapters map to trust dimensions:
- genesis adapter → T (tile_proof at issuance)
- attestation adapter → A (chain depth + third-party witnesses)
- redaction adapter → S (flag binding persistence)
- gossip adapter → G (liveness beacon)
